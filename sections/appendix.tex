\section{Supplementary Route Results}
\label{app:route_robustness}
In all route tables, Route~1 is natural-language reasoning, Route~2 is
natural-language simulation over a code trace, and Route~3 is deterministic code
execution. Captions and result paragraphs state whether a table uses the full
evaluation set, a 25\% subset, a single seed, or per-arm parse-normalized
denominators.

\subsection{Complexity-Stratified Route Accuracy}
\label{app:complexity_stratified_route_accuracy}

We stratify the main route-accuracy evaluation by the asymptotic complexity
class of each task. Across classes, Route~3 remains the main source of
improvement over Route~2, while Route~2 is close to Route~1.

\appendixresult{Accuracies and deltas in \cref{tab:appendix_complexity_routes} are percentages
or percentage points. McNemar tests are exact two-sided tests on paired outcomes.
The main route pattern is not an artifact of averaging together easy and hard
task families: the Route~3--Route~2 gain remains the largest observed change in
the high-support classes, while Route~2 stays close to Route~1.}

\paragraph{Task mapping.}
The largest strata are $O(n^2)$ dynamic-programming, sorting, string, graph, and
shortest-path tasks; and NP-hard ILP, graph coloring, edge-disjoint paths,
shortest path, and TSP tasks. The smaller strata cover constant-time arithmetic
and geometry, logarithmic binary search, linear selection/string matching,
$O(n \log n)$ scheduling/sorting/hulls, $O(n^2 \log n)$ Kruskal MST, and cubic
dynamic-programming/shortest-path routines.

\section{Reproducibility Notes}
\label{app:reproducibility_notes}

The appendix tables are generated by the accompanying analysis code.

\paragraph{Route tables.}
\path{analyze_route_accuracy_tables.py} uses the full six-model route evaluation
and paired route outcomes for McNemar tests.

\paragraph{Translation shot ablation.}
\path{analyze_translation_shot_ablation.py} generates the matched 25\% subset
sweep for rows 0--5; the 10-shot row is retained as the original
main-evaluation reference.

\paragraph{Additional evaluators.}
\path{analyze_rlm_subset_results.py} covers the 25\% subset recursive-language
model runs, \path{analyze_coding_model_table.py} covers the coding-specialized
model rows, and \path{analyze_frontier_nopatch_table.py} covers the 350-instance
frontier controlled subset.

\paragraph{Failure modes.}
\path{analyze_code_failure_distribution.py} reports Route~3 failures after
excluding parse-only rows.
